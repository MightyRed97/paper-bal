Abstract (publisher summary):
Multi-cloud environments have become common as companies aim to prevent cloud
vendor lock-in for security and cost concerns, and microservice architecture is often
adopted for its flexibility. Combining multi-cloud with microservices creates the
problem of routing requests among all possible microservice instances distributed
across multiple clouds. This paper presents a learning-based approach to route
requests in order to balance the load, in which the performance of each microservice
is modeled explicitly through machine learning models. The model derives response
time from request volume, routing decisions, and other cloud metrics; a balanced
routing decision is then obtained by optimizing the model with Bayesian Optimization.
With this approach, the request routing decision can adapt to dynamic runtime metrics
instead of remaining static across all circumstances. Experiments show the proposed
learning-based dynamic load balancing outperforms conventional static strategies in
terms of response time and load distribution.
